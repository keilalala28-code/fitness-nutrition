# 食物营养数据更新指南

## 快速更新步骤

### 1. 打开数据文件
编辑 `data/foods.json` 文件

### 2. 添加新品牌
在 `brands` 对象下添加：

```json
"新品牌名": {
  "category": "drink",  // 分类：drink/takeout/fastfood等
  "updateDate": "2026-03",
  "products": [
    {
      "id": "unique-id",
      "name": "产品名称",
      "servingSize": 500,
      "servingUnit": "一杯约500ml",
      "nutrients": {
        "calories": 100,
        "protein": 2.0,
        "carbs": 15.0,
        "fat": 3.0
      },
      "tags": ["标签1", "标签2"]
    }
  ]
}
```

### 3. 添加新产品到已有品牌
找到对应品牌的 `products` 数组，添加新对象即可。

### 4. 验证数据
```bash
npm run build
```
如果构建通过，说明数据格式正确。

## 分类说明 (category)

| 值 | 含义 | 示例 |
|---|------|------|
| `drink` | 饮品 | 奶茶、咖啡、果汁 |

| `vegetable` | 蔬菜 | 西兰花、菠菜 |
| `fruit` | 水果 | 苹果、香蕉 |
| `snack` | 零食 | 坚果、蛋白棒 |
| `grain` | 谷物 | 燕麦、全麦面包 |

## 营养数据来源

### 优先级
1. **品牌官方公示** - 小程序、官网、包装标签
2. **第三方权威检测** - 上海消保委等
3. **《中国食物成分表》** - 基础食材
4. **外卖平台数据** - 美团/饿了么商家标注

### 常用查询渠道
- 霸王茶姬：官方小程序 > 产品详情
- 喜茶/奈雪：公众号 > 营养公示
- 瑞幸：App > 产品详情页
- 即食鸡胸肉：包装背面营养成分表

## 注意事项

1. **ID唯一性**：每个产品的 `id` 必须全局唯一
2. **数值单位**：
   - calories: kcal (每100g/ml)
   - protein/carbs/fat: g (每100g/ml)
   - servingSize: g 或 ml
3. **营养数据**：统一换算为每100g/100ml的含量
4. **更新日期**：修改数据时更新 `updateDate` 和顶部 `_meta.lastUpdated`

## 常见问题

### Q: 为什么要统一换算成每100g？
A: 方便用户输入任意克数时自动计算，比如输入150g自动得出1.5倍营养。

### Q: 新品没有官方数据怎么办？
A: 可以参考同品牌类似产品估算，并在 tags 中添加"估算"标签。

### Q: 数据有误怎么修改？
A: 直接在 `foods.json` 中找到对应产品修改 `nutrients` 数值即可。
